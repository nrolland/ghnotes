Dynamic programming

Dynamic programming is equivalent to the following two concepts used together : recursion and deforestation

Recursion

Recursion allows to express solutions that bends itself to it in a simpler way. It is merely a way to formulate a solution in terms of a subproblem graph of diminishing size. by itself, it does not reduce the complexity of computing the solution.

Deforestation

If subproblems appears multiple times in the subproblem graph, then storing intermediary results allows for a dynamic folding of it, replacing the graph with its computed value. This is where the improvement of the dynamic programming comes from.

Two strategies are possible for this :

  • a bottom up approach, evaluating adjacents sub problems in a reverse topological order, that is tackling simpler problem first and guaranteing that simpler problems are all solved before increasing complexity. This solution has low overhead but special care must be taken to the order of evaluation

  • a top down approach, where we add a memoization steps, and that keep the natural. This allows unnessary branch, if any, to not be computed.

Both approaches yield the same asymptotical performance.

Those examples are taken from the book Introduction to algorithm.

Rod cutting

  • simple recursion
1: //rod cutting recursive - inefficient
2: let price = [|1;5;8;9;10;17;17;20;24;30|]
3: 
4: //the maximum price for a size n is the maximum among i=1..n of 
5: //(selling an entire piece if size i + the maximum price of an optimal subdivision for the remaining part) 
6: let rec rodcutting_rec  = function | 0 -> 0 | n -> Seq.init n (fun i -> price.[i] + rodcutting_rec (n-i-1)) |> Seq.max 
7: [1 .. 10] |> List.map (fun i -> i, rodcutting_rec i)
  • bottom up
1: //rod cutting recursive - dynamic programming
2: //the layout of the ordering enforces access to stored value, and folding of the computation tree
3: let rodcutting_bu n = 
4:     let r = Array.init (price.Length + 1) (fun _ -> 0)
5:     let rec up k  = r.[k] <- Seq.init k (fun i -> price.[i] + r.[k-i-1]) |> Seq.max 
6:                     if k < n then up (k+1) else r.[k]
7:     up 1    
8: [1 .. 10] |> List.map (fun i -> i, rodcutting_bu i)
  • memoized recursion
 1: //rod cutting recursive with memoization - dynamic programming
 2: //the memoization of each execution ensure the value storage without particular ordering
 3: let memoize f = 
 4:     let d =  System.Collections.Generic.Dictionary<_, _>()
 5:     d, fun n -> if d.ContainsKey n then d.[n] 
 6:                 else d.Add(n, f n);d.[n]
 7: 
 8: let rodcutting_dp n =
 9:     let rec rodcutting_rec  = function | 0 -> 0 | n -> Seq.init n (fun i -> price.[i] + rodcutting_rec (n-i-1)) |> Seq.max  
10:     let _, mf = memoize rodcutting_rec
11:     mf n
12:     
13: [1 .. 10] |> List.map (fun i -> i, rodcutting_dp i)

Matrix product

  • simple recursion
 1: //rod cutting recursive - inefficient
 2: let p = [|30;35;15;5;10;20;25|]
 3: 
 4: let tee f x = f x;x
 5: 
 6: //the cost of n*p . p*q is n*p*q   
 7: //the minimum cost parenthesizing is the minimum for i = 1..n
 8: //the min cost of parenthesizing 1..i together + (i+1).. together + multiplying both parts 
 9: let rec matrixproduct_rec(i,j) = //assume 1 <= i,j <= n
10:     match j - i with 
11:     | n when n <= 0 -> 0 
12:     | n -> Seq.init n (fun k -> matrixproduct_rec(i,i+k) + p.[i-1]*p.[i+k]*p.[j] + matrixproduct_rec(i+k+1,j)) |> Seq.min  |> tee (printfn "%A,%A" (i,j))
13: 
14: let product xs ys = [ for x in xs do for y in ys do yield x,y ]
15: product [1 .. 6] [1 .. 6] |> List.map (fun i -> i, matrixproduct_rec i)
  • bottom up
 1: module Seq = let inline minDef def items = if items |> Seq.isEmpty then def else items |> Seq.min
 2:     
 3: //we order the problem by the size of the subproblem j - i and solve problem of increasing size
 4: // i j refers to the index of the matrix to be multiplied (1 based)
 5: let matrixproduct_bu(ifinal,jfinal) = 
 6:     let m = Array2D.init p.Length p.Length (fun _ _ -> 0)
 7:     let pbsizetotal = (p.Length-1)
 8:     let rec up pbsize = 
 9:         // we have (pbsizetotal - n) pbs of size n, and the index are i, j / j - i = n, and max j <= p.length
10:         Seq.init (pbsizetotal-pbsize) ((+)1>> fun i-> i,i+pbsize) |> Seq.iter(fun(i,j) -> 
11:             m.[i,j] <- (Seq.init pbsize (fun k -> // for each possible split at i + k (k is 0 based offset)
12:                         m.[i,i+k] + p.[i-1]*p.[i+k]*p.[j] + m.[i+k+1,j])  //we compute the cost, accessing only pb of size  k < n and n - k - 1 < n //|> tee (printfn "i,j %A, k %A, %A" (i,j) (k))
13:                         |> Seq.minDef 0  )) // and take the min
14:         if pbsize < (jfinal-ifinal) then up (pbsize+1) else m.[ifinal,jfinal]
15:     up 0
16:     
17: product [1 .. 6] [1 .. 6] |> List.map (fun i -> i, matrixproduct_bu i)
